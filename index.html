Mechanic

The MPI task management system for the dynamical astronomy

Download this project as a .zip file Download this project as a tar.gz file

Overview

The aim of the Mechanic is to help in conducting massive numerical simulations. It provides tools to help you focus on numerical problem: the storage, checkpoints (backups, restarts), setup, command-line, splitting the workload, CPU-communication etc. is handled by the core code. The rest is in user-supplied module.

Key features

  • Core-module architecture -- the numerical code separated from the communication and storage layers
  • C-based user-supplied modules, which allows to connect several C-interoperable programming languages (such as C++, OpenCl, CUDA, partly Fortran2003+)
  • Task farm communication pattern (on top of the MPI2 standard), working well on large CPU-clusters, PC-workstations or laptops
  • Unified data management (on top of the HDF5 library) with storage scheme implemented through the rich user API
  • The module setup implemented through the API (among with the command-line options)
  • No MPI or HDF5 knowledge required
  • Pool-based task farm, task pools created on-the-fly, based on data from previous pools (example: Genetic Algorithms)
  • Several storage types, which allows to store result data for different purposes (example: Dynamical Maps)
  • Custom storage scheme per pool

Sample module

#include "MMechanic2.h"

int Storage(pool *p) {
  p->task->storage[0].layout = (schema) {
    .path = "conditions";
    .dataspace_type = H5S_SIMPLE;
    .datatype = H5T_NATIVE_DOUBLE;
    .rank = 2;
    .dim[0] = 5;
    .dim[1] = 6;
    .use_hdf = 1;
  return 0;
}

int TaskProcess(pool *p, task *t, setup *s) {
 t->storage[0].data[0][4] = 123.45;
 return 0;
}
